## **1. 서킷 브레이커란?**  
서킷 브레이커(Circuit Breaker) 패턴은 장애가 발생하거나 요청 실패율이 높은 경우, 특정 요청을 자동으로 차단하여 시스템의 전체 장애를 방지하는 설계 패턴입니다.  
이 패턴은 마치 전기 회로 차단기처럼 동작하며, 과부하가 걸린 시스템이 정상적인 상태로 복구될 시간을 가질 수 있도록 도와줍니다.

### **서킷 브레이커의 주요 개념**
- **Closed(닫힘) 상태**: 정상적으로 요청을 처리하는 상태.
- **Open(열림) 상태**: 일정한 실패율이 감지되면 서킷이 열려 새로운 요청을 차단.
- **Half-Open(반열림) 상태**: 일정 시간이 지난 후, 일부 요청을 허용하여 시스템이 정상적으로 복구되었는지 테스트.

### **서킷 브레이커의 주요 기능**
- 과부하가 걸린 시스템을 보호하여 장애 전파 방지
- 장애 발생 시 빠르게 대체 서비스나 캐시된 데이터를 제공하여 사용자 경험 유지
- 자동 복구 기능을 통해 정상 상태로 빠르게 복귀

## **2. 대규모 트래픽으로 인한 서버 과부화 해결 방법**  
대규모 트래픽으로 인해 서버가 과부하될 경우, 다음과 같은 최적화 방법을 활용할 수 있습니다.

### **1) 불필요한 콘텐츠 제거**
- 서버에서 전송하는 데이터를 최소화하여 트래픽 감소
- 예: API 응답에서 불필요한 필드 제거, 불필요한 이미지 및 애니메이션 줄이기  

### **2) CDN(Content Delivery Network) 활용**
- 정적 파일(이미지, CSS, JS)을 CDN에 배포하여 서버 부하 감소
- 글로벌 사용자를 위해 지리적으로 가까운 CDN 서버에서 빠르게 제공  

### **3) 콘텐츠 캐싱**
- 자주 사용되는 데이터를 캐싱하여 DB 및 서버 부하 감소
- 브라우저 캐싱, 프록시 캐싱, 서버 캐싱(Redis, Memcached) 등을 활용  

### **4) 콘텐츠 압축**
- 네트워크 대역폭 절약을 위해 Gzip, Brotli 같은 압축 기법 사용
- 이미지(WebP, AVIF) 및 동영상 최적화  

### **5) 콘텐츠의 우아한 저하(Graceful Degradation)**
- 서버 부하가 심할 때 일부 기능을 제한적으로 제공
- 예: 고해상도 이미지 대신 저해상도 이미지 제공, 실시간 기능 일시 중단

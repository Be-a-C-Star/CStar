

## 데이터베이스란?

\*\*데이터베이스(Database)\*\*는 **정보를 모아서 정리해 놓은 저장소**예요. 쉽게 말해, **많은 데이터를 체계적으로 보관하고 쉽게 꺼내 쓸 수 있게 만든 상자**

예시:
학교에서 학생들의 이름, 학번, 성적 등을 모아둔 학사 시스템도 하나의 데이터베이스

### 데이터베이스의 특징

- **중복을 줄여요**: 같은 데이터를 여러 번 저장하지 않는다.
- **빠르게 찾을 수 있다**: 필요한 정보를 쉽게 검색할 수 있다.
- **여러 사람이 함께 써요**: 동시에 여러 명이 접근해도 문제 없음
- **안정적으로 저장해요**: 갑자기 꺼져도 데이터가 잘 보존

---

## 엔터티(Entity)란?

**엔터티**는 현실 세계에서 **데이터로 저장할 수 있는 것들**

예시:

- 학생(Student)
- 도서(Book)
- 주문(Order)

즉, 데이터베이스 안에 **무언가를 저장할 대상**이 바로 엔터티

---

##  강한 엔터티와 약한 엔터티

###  강한 엔터티(Strong Entity)

- **혼자서도 존재 가능**.
- \*\*고유한 키(기본 키)\*\*가 있음.

예: 학생(Student)은 학번이라는 키로 구분되므로 강한 엔터티

###  약한 엔터티(Weak Entity)

- **다른 엔터티에 의존**
- **자기만의 키가 없음**
- 보통 \*\*외래 키(Foreign Key)\*\*를 이용해서 강한 엔터티에 연결

예: 주문 상세(Order Detail)는 주문(Order)이 있어야만 의미가 있으므로 약한 엔터티

---

##  릴레이션(Relation)이란?

릴레이션은 **엔터티들 사이의 관계**

예시:

- 학생(Student) ↔ 수강신청(Course Registration)
- 고객(Customer) ↔ 주문(Order)

릴레이션은 **표 형태의 테이블**로 표현. 그래서 데이터베이스를 **관계형(Relational) 데이터베이스**라고도 함.

---
## 속성(Attribute)과 도메인(Domain)

### 속성(Attribute)

속성은 **엔터티가 가진 정보의 항목**

예시:

- 학생 엔터티의 속성: 이름, 학번, 전공 등
- 책 엔터티의 속성: 제목, 작가, ISBN 등

### 도메인(Domain)

속성이 가질 수 있는 **값의 범위**
예:

- '성별'이라는 속성의 도메인: 남, 여
- '나이'의 도메인: 0 \~ 150

---

## 필드(Field)란?

**필드**는 테이블에서 \*\*하나의 칸(셀)\*\*
즉, 특정 속성에 저장된 **하나의 데이터 값**

예시:

| 이름   | 학번    | 전공       |
| ------ | ------- | ---------- |
| 홍길동 | 2023001 | 컴퓨터공학 |

여기서 "홍길동", "2023001", "컴퓨터공학" 각각이 **필드**

---

## 레코드(Record)란?

**레코드**는 테이블에서 \*\*하나의 행(Row)\*\*
즉, **하나의 엔터티에 대한 모든 정보**를 모아놓은 줄

예시:

| 이름   | 학번    | 전공       |
| ------ | ------- | ---------- |
| 홍길동 | 2023001 | 컴퓨터공학 |

이 한 줄 전체가 하나의 **레코드**

---

## 타입(Type)

**타입**은 각 속성이나 필드에 저장될 **데이터의 종류**

예시:

- 문자열 타입: 이름(홍길동), 전공(디자인)
- 숫자 타입: 나이(23), 학번(2023001)
- 날짜 타입: 생일(2001-02-20)

---

## 키워드 정리

| 용어         | 의미                                   |
| ------------ | -------------------------------------- |
| 데이터베이스 | 정보를 모아둔 체계적인 저장소          |
| 엔터티       | 저장할 대상(학생, 책 등)               |
| 속성         | 엔터티가 가진 정보 항목(이름, 나이 등) |
| 도메인       | 속성이 가질 수 있는 값의 범위          |
| 필드         | 하나의 데이터 값 (셀)                  |
| 레코드       | 한 엔터티에 대한 전체 데이터 (행)      |
| 릴레이션     | 엔터티 간의 관계                       |
| 타입         | 데이터의 종류 (문자열, 숫자 등)        |
| 강한 엔터티  | 혼자 존재 가능, 기본 키 가짐           |
| 약한 엔터티  | 다른 엔터티에 의존, 기본 키 없음       |

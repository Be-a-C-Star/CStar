# MySQL
- 오픈 소스 관계형 데이터베이스 관리 시스템(RDBMS)
- Structured Query Languages(SQL)을 사용하여 데이터 관리
- Oracle이 소유하고 있지만, 무료 버전도 많이 쓰임

## MySQL 특징
1. 관계형 데이터베이스: 데이터를 테이블에 구조화해서 저장
2. SQL 사용: 데이터를 조회, 삽입, 수정, 삭제하기 위해 SQL 사용
3. 오픈 소스: 누구나 무료로 사용할 수 있고, 커뮤니티가 활발

## MySQL을 사용하는 이유
- 웹 애플리케이션에 적합
- 빠른 성능과 안정성
- 쉬운 사용법과 관리 도구

##  MySQL 기본 구조
- Database: 여러 개의 테이블을 포함
- Table: 행(row)과 열(column)로 구성된 데이터 집합
- Row: 실제 데이터 한 줄
- Column: 각 데이터 속성 (이름, 나이 등)

## MySQL 핵심 개념
1. INDEX
  - 개념: 특정 컬럼에 인덱스를 걸면, 그 컬럼을 기준으로 검색 속도가 훨씬 빨라짐.
  - 특징: 너무 많은 인덱스는 오히려 성능을 해칠 수 있음.

2. JOIN
  - 개념: 두 개 이상의 테이블을 연결해서 하나처럼 데이터를 조회하는 것.

3. Transaction
  - 개념: 하나의 작업 단위. 여러 쿼리를 묶어서 실행하고, 모두 성공해야 실제로 반영됨. 중간에 오류가 나면 전체를 ROLLBACK.

4. 뷰 (View)
  - 개념: 자주 쓰는 SELECT 문을 가상 테이블처럼 저장해서 마치 테이블처럼 사용.

5. 정규화 (Normalization)
  - 개념: 데이터 중복을 줄이고, 논리적으로 정리된 구조를 만들기 위한 과정. 테이블을 나누고, 관계를 통해 연결함.

# ERD (Entity Relationship Diagram)
어떤 데이터를 저장할 때, 이걸 어떻게 나눠서 저장할지, 그리고 서로 어떤 관계가 있는지 시각적으로 그려놓은 그림
| 개념                | 뜻              | 예시                                 |
| ----------------- | -------------- | ---------------------------------- |
| Entity (엔티티)      | 저장할 대상 (테이블)   | `User`, `Post`, `Comment`          |
| Attribute (속성)    | 그 대상의 정보들 (컬럼) | `User` 엔티티의 `name`, `email`, `age` |
| Relationship (관계) | 엔티티끼리 연결       | `User`가 `Post`를 작성한다 (1\:N)        |

### Relationship (관계)

| 관계   | 설명        | 예                                         |
| ---- | --------- | ----------------------------------------- |
| 1:1  | 한 명당 하나   | 주민등록증 - 사람                                |
| 1\:N | 한 명이 여러 개 | 사용자 - 게시글                                 |
| N\:M | 서로 여러 개   | 게시글 - 태그 (하나의 글에 여러 태그, 하나의 태그는 여러 글에 있음) |

## ERD가 중요한 이유
- DB 구조를 미리 설계하고 공유할 수 있음
- 개발자, 디자이너, 기획자가 같은 그림을 보며 얘기 가능
- 개발 중에 테이블 구조 변경할 일이 줄어듦
